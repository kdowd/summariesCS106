Project Summary: Employee Management System

The Employee Management System is a desktop application built using WPF, C#, and .NET 8.0, with MongoDB Atlas as the database. It's designed to help a company manage its employees efficiently, catering to different roles like Admins, Managers, HR, and regular Employees. So far, we've implemented core features like adding, viewing, editing, and deleting employee records (CRUD operations), with role-based access (e.g., Admins see more details than Guests). We've also added login functionality, connected to MongoDB Atlas for data storage, and set up a tabbed interface in the main window (Employees, Dashboard, Leave Requests, Payroll, Admin). Currently, we're working on additional features like expanded user roles, leave requests, and payroll management, ensuring the system is secure, user-friendly, and scalable.

User Stories (Non-Technical Language):

  • As a new user, I want to log in with my username and password so that I can access the system securely. When I open the app, I see a login window where I can type my username (e.g., "admin") and password (e.g., "admin123"). If I enter the correct details, I’m taken to the main screen with tabs like "Employees" and "Admin." If I click "Cancel," I can still use the app as a Guest but with limited access.

  • As a Guest user, I want to view a limited list of employees so that I can see who works here without accessing sensitive details. If I skip the login by clicking "Cancel," I enter the app as a Guest. In the "Employees" tab, I see a table with just the Name and Position of employees—no Contact or Role details. Restricted tabs like "Admin" and "Payroll" are hidden from me.

Manager only:

  • As a Manager, I want to edit an employee’s details so that I can fix mistakes or update their information. In the "Employees" tab, I select an employee (e.g., "John Doe") from the list and click an "Edit" button. I can change details like their Position (e.g., from "Developer" to "Senior Developer") or Contact, then save. The updated details show up in the list right away.

HR only:

  • As an HR person, I want to delete an employee who has left the company so that our records stay accurate. In the "Employees" tab, I select an employee who’s no longer with us (e.g., "Jane Smith"), click a "Delete" button, and confirm the action. The employee disappears from the list, ensuring our records only show current staff.

Admins only:

  • As an Admin, I want to see a list of all employees so that I can manage the team effectively. After logging in as an Admin, I go to the "Employees" tab and see a table with everyone’s details, like their ID, Name, Position, Contact, Role, and Date of Birth. This helps me keep track of who’s on the team and their roles.

  • As an Admin, I want to add a new employee to the system so that we can keep our records up to date. In the "Employees" tab, I click a button to add a new employee. I fill in their details—like Name (e.g., "Sarah Lee"), Position (e.g., "Designer"), Contact, Username, Password, Role, and Date of Birth—and save it. The new employee then appears in the list, and I know the system has updated.

  • As an Admin, I want to see different tabs based on my role so that I can access the tools I need. After logging in as an Admin, I see all tabs in the main window: Dashboard, Employees, Leave Requests, Payroll, and Admin. I can switch between them to manage employees, check payroll, or handle admin tasks, knowing my role gives me full access.

Project Summary: Restaurant Ordering and Management Workflow.

This system is designed to streamline the entire restaurant ordering and management workflow, it allows customers to browse the menu, customize and place orders, while giving staff real-time visibility into sales, order statuses, access to update menu, add promotions, and get insights.

User Stories (Non-Technical Language):

  • As a Customer, I want to view the full menu with category filters and sort by popularity so that I can quickly find what I'm most interested in.

  • As a Customer, I want to add, update, or remove items in my cart and then proceed to checkout so that I can complete my order accurately and efficiently.

  • As a Customer, I want to see my past orders and re-order with a single click so that I can quickly repeat my favorite meals.

  • As a Customer, I want to mark menu items as favorites so that I can access them easily in future sessions.

  • As a Customer, I want a dedicated “Special Deals” category so that I can discover and take advantage of promotions.

Admins only:

  • As an Admin/Staff, I want to see total sales for today, this week, and this month so that I can monitor revenue performance at a glance.

  • As an Admin/Staff, I want to add, update, or remove menu items (including availability) so that the system always reflects our current offerings.

  • As an Admin/Staff, I want to export sales reports to CSV so that I can perform deeper analysis or share data with other stakeholders.

Project Summary: Rallypoint

Rallypoint is a tech startup located in Auckland NZ and has contracted you to develop their app for initial release.

Primarily focused on event promotion they have given you a list of must have features for the app which are;

  • Users must be able to view all events posted globally.

  • Access to social media aspects including chats and video and image interaction without requiring further verification than the initial account setup.

The second part of the required features includes a semi automated verification system for hosts, which will determine whether or not a user is eligible to become a verified host and begin posting events.

This verification should include at least one of the following, Identity and/or Location verification.

And finally the app should be deployable across mobile platforms either ios and or android, along with web and potentially desktop compatibility. Apart from these mandatory features Rallypoints executives have given you developmental discretion in regards to other features of the application.

As this is a startup company Rallypoint has requested that dev costs and hosting should be kept to a minimum and therefore have chosen to contract you as they believe you will be able to create a functioning app that is set up for scaling right from the start all while maintaining budget friendly.

User Stories (Non-Technical Language):

  • User A - Will sign up for an account and complete profile setup either through the app create account page or through the verification portal.

  • User A - Now has access to standard user features including access to the map view, chat, explore and profile pages.

  • User B - Has completed online verification and now navigates to the add event option from the profile page and can now post their first event, and has added user A to a group chat where that event is now shared to user A.

  • User C - Has attended User B's event and has taken some photos and videos of said event and now from the explore page has posted these photos and videos tagging User B

  • User A - Has now seen User C's post on the explore page and has decided to like the post.

  • User B - Has also seen the post and has decided to follow user b back along with like the post and post a comment.

Project Summary: Author Portfolio Website.

A React-based web application for an independent author to showcase and sell their published and upcoming books. The site features a library-themed design with integrated e-commerce functionality, allowing visitors to browse books, view artwork, and make purchases seamlessly.

Author Name: Kia Beniston Tech Stack: React frontend, Supabase database, Stripe payment processing, Clerk authentication. Key Features: Interactive bookshelf landing page Book preview on hover Integrated purchasing system Artwork gallery with print sales Responsive design optimized for minimal scrolling and maximum conversion

User Stories (Non-Technical Language):

  • General Visitors & Customers As a potential reader visiting the website, I want to see an attractive library-themed homepage that immediately shows me what books are available, so I can quickly decide if I'm interested in the author's work.

  • As someone browsing the site, I want to hover over books on the virtual bookshelf and see a brief description pop up, so I can get a taste of each story without clicking through multiple pages.

  • As a book buyer, I want to click on a book and be taken directly to a purchase page where I can buy it with just a few clicks, so the buying process is quick and straightforward.

  • As an art enthusiast, I want to browse a dedicated artwork section where I can view and purchase small prints of illustrations related to the books, so I can own physical pieces of the creative work.

  • As a curious visitor, I want to easily find and read the author's biography and contact information on a simple about page, so I can learn more about who wrote these books.

  • As a mobile user, I want the entire website to work smoothly on my phone without excessive scrolling, so I can browse and buy books easily while on the go.

  • As a returning visitor, I want to see placeholder spaces for upcoming books alongside published ones, so I know what new content to expect and can stay engaged with the author's work.

  • As someone making a purchase, I want a secure and trusted payment system that protects my financial information, so I feel confident buying books online.
